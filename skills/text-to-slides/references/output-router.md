# Output Router

Use both project skills as complementary output routes:

- `text-to-ppt`: editable PowerPoint `.pptx` for business decks, Google Slides upload, and manual editing.
- `text-to-slides`: HTML slide/card-news output for Remotion animation, video production, and web previews.
- `text-to-slides-video`: motion-slide video scene planning, Remotion handoff, and GIF motion previews from text.
- `text-to-gif`: short looping animated GIFs for metric highlights, chart motion previews, and lightweight social proof.

Default routing:

- User says `PPT`, `PowerPoint`, `Google Slides`, `deck file`, or needs editing: use `text-to-ppt`.
- User says `slides`, `HTML`, `Remotion`, `video`, `animation`, `card news`, or social carousel: use `text-to-slides`.
- User says `slides video`, `motion slide video`, `text to video`, or wants a video-ready slide sequence: use `text-to-slides-video`.
- User says `GIF`, `looping`, `animated image`, `quick motion preview`, or wants a compact chart animation: use `text-to-gif`.
- User asks for multiple outputs: create one shared content plan, then export PPTX, HTML, and/or GIF from the same story spine.
